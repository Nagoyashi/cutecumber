# v0.2.0 — Production hardening

Phase 4 of cutecumber.cc: close out the high/medium-priority post-launch audit
and prove that the things a live site can't be wrong about — database backups,
password-reset email, legal pages — actually work in production.

No user-facing feature changes. This release is about the parts you only notice
when they fail.

## Hardening

- **Database backups are live.** SQLite is now streamed off-machine to object
  storage (Cloudflare R2, EU jurisdiction) via Litestream, so losing the Fly
  machine no longer loses the data. (#9)
- **Password-reset email verified in production.** Confirmed the reset flow
  delivers mail end-to-end on the live site. (#4)
- **Legal pages completed.** Every placeholder in the imprint and privacy
  policy is filled in for the live EU site. (#3)

## Security

- **Regression tests for the security guarantees.** Automated tests now cover
  CSRF protection, IDOR (object-ownership) checks, and the cookie-free
  guarantee, so a future change can't silently break them. (#6)
- **Dependency vulnerability scanning + reproducible builds.** CI scans
  dependencies for known vulnerabilities and the container build is digest-pinned
  for reproducibility. (#7)

## Notes

- No schema or API changes; deploy is a standard `fly deploy`.
- Dependabot runtime bumps (`actions/checkout` v7, `actions/setup-python` v6,
  python 3.14-slim, gunicorn 26) are deferred to v0.2.1.

---

**Build story (raw material — owner supplies voice + facts):** this cycle was a
post-launch audit burndown — five issues, mostly the unglamorous "is the safety
net actually attached?" kind. The one with teeth was backups: production had
been running with no database backup at all (the replica was never configured),
so "confirm backups" became "set them up." Wiring Litestream to an EU R2 bucket
surfaced a real gotcha — Litestream's bare `s3://` URL form can't set the S3
region, so it fell back to an AWS region lookup that R2 rejects with
`InvalidAccessKeyId`; the fix was a config file pinning `region: auto`. Verified
by streaming a live snapshot to R2 before shipping.
