# v0.2.1 — Maintenance & polish

A patch release: the runtime bumps deferred from v0.2.0 plus the remaining
low-priority cleanups from the post-launch backlog. No user-facing feature
changes, no schema changes — deploy is a standard `fly deploy`.

## Dependencies

- **Runtime bumps** that were held back from v0.2.0, now landed together:
  `actions/checkout` v7, `actions/setup-python` v6, the Python base image to
  `3.14-slim`, and gunicorn to 26. CI's `python-version` moved to 3.14 in
  lockstep with the Docker base, per the lockstep rule. (#18, #19, #20, #22)

## Security

- **Password-reset tokens stay out of access logs.** The reset link carries a
  single-use token in its URL path; gunicorn's access logger now redacts it
  before writing each line, closing an avoidable replay window. (#11)

## Performance

- **Curated art is immutable-cached.** Avatar and decoration-pack SVG tiles now
  carry long-lived `immutable` cache headers (like the fonts already did), so
  public pages stop revalidating unchanging art on every load. An art swap
  ships a new filename, so cached URLs never go stale. (#12)

## Operations

- **Optional error reporting.** When `ERROR_WEBHOOK_URL` is set, unhandled 500s
  are POSTed as JSON (method + reset-token-scrubbed path + traceback) to any
  JSON sink — a Sentry ingest URL, a Slack/Discord webhook, a self-hosted
  collector. Unset = no-op; no new dependency (stdlib only, like the mailer). (#13)

## Docs

- **Comments and docs reconciled with shipped reality.** Fixed the `schema.sql`
  `avatar_kind` comment (now `emoji | gradient | image | set`) and its
  DECISIONS cross-reference, the stale "manual-by-email" account-deletion note,
  and the Python-version references left at 3.12. (#10)

---

**Build story (raw material — owner supplies voice + facts):** the tidy-up
release. Most of it was the kind of work that never shows up on the page:
merging the held dependency bumps, fixing comments that had drifted from the
code, and caching art that should always have been cached. Two bits had a bit
of craft to them. The reset-token-in-logs fix is layered correctly — a pure
scrubber in `security.py` (unit-tested) that gunicorn's access logger calls over
every request path, rather than a regex bolted onto the log format. And error
monitoring went in the house way: instead of pulling the Sentry SDK and its
transitive tree for what is fundamentally "POST some JSON when something
breaks," it's a dozen lines of stdlib `urllib` behind an env var — the same
reasoning that kept the mailer SDK-free. The whole thing stays inside the
fixed-stack and zero-JS rules; the test suite went from 90 to 97.
