# v0.4.0 — Design refresh v2

A visual refresh of the two surfaces you actually live in: the public **landing
page** and the logged-in **dashboard/editor**. Same kawaii-minimal brand, same
hard rules — just less plain-and-centered, more cute-and-alive.

No new trackers, no cookies for anonymous visitors, still zero JavaScript and
zero third-party requests on the public page. The landing is **4.7 KB gzipped**
(budget is 15 KB), and every button still clears WCAG-AA contrast.

## Landing

- **A real hero.** Split layout — copy on the left, a live example-page preview
  card on the right — with a top bar and a four-up feature grid. (#40)
- **An ambient brand background.** A soft cucumber-slice watermark and a dozen
  little floating motifs fill the desktop margins, all from one inline SVG
  sprite, animated with CSS only and gently disabled for anyone who prefers
  reduced motion. (#41)

## Dashboard

- **Glassy, wider, calmer.** Frosted cards, a proper top bar with nav pills, a
  "your page is live" bar with the live URL plus copy-link and visit buttons,
  and the same ambient background as the landing. (#42)
- **The drawn avatars lead.** The profile picker is now the 12 hand-drawn
  character tiles plus a photo upload; new accounts start on a little sprout
  instead of the cucumber emoji. (#43)

## Accessibility

- The accent pills carry a dark ink rather than white text — white-on-pink was
  2.36:1 (below AA). A token + test guards this across the landing and dash.

## Notes

- **No data migration.** Existing avatars are untouched: freeform emoji and flat
  gradients simply left the *picker*, but still render, and anyone who already
  has one keeps it (the editor shows it as a "current" tile). New accounts
  default to `set:sprout`. (DECISIONS #13 addendum)
- No schema or API changes; deploy is a standard `fly deploy`.
- Auth screens and public profile pages keep their current look — their refresh
  is the next cycle.

---

**Build story (raw material — owner supplies voice + facts):** a designer
handoff arrived as React/Babel prototypes — explicitly "don't ship this, rebuild
it in your stack." So the whole refresh was re-implemented as plain Jinja
templates and vanilla CSS: the ambient background became a single inline-SVG
`<symbol>` sprite referenced by `<use>` (so a dozen motifs cost almost nothing
and gzip to nothing), and the React component tree became native `<details>`
sections and radio-button pickers that work with JavaScript switched off. Two
things were caught on the way in: the prototype re-introduced the exact
white-on-pink contrast failure a previous release had just fixed (kept the dark
ink), and "retire emoji/gradient avatars" was quietly a breaking change for
existing users (kept render support so nobody's page changes under them). The
landing's headline claim — "about two kilobytes" — still holds: it's the user's
public page that's tiny, and that page wasn't touched this cycle.
